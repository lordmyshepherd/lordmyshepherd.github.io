1.Position속성 -relative, absolute, fiexd

​

position속성을 왜 사용하는가?

: css의 position property를 이용하면 html 코드와 상관 없이 원하는 위치에 요소를 넣을 수 있다.

1)relative

: position: relative; 자체로는 특별한 의미가 없습니다.

딱히 어느 위치로 이동하지는 않습니다.

위치를 이동시켜주는 top, right, bottom, left 프로퍼티가 있어야 원래의 위치에서 이동할 수 있습니다.

​

Q. 아래의 각 element의 원래 위치가 어디인가?

첫번째 빨간점 = div의 원래 위치

두번째 빨간점 = div.relative의 원래 위치

세번째 빨간점 = div.relative.top-20의 원래 위치

아래 코드를 css에 추가해주면 div.relative.top-20 element가 세번째 빨간점을 기준으로 이동하게 되는 것이다.

.top-20 {

top: -20px;

left: 30px;

}

​

relative 속성

2)absolute

: position: absolute; 는 이름과 같이 절대적인 위치에 둘 수 있다. 어떤 기준으로 절대적이냐 하면, 특정 부모에 대해 절대적으로 움직이게 된다.

부모 중에 position이 relative, fixed, absolute 하나라도 있으면 그 부모에 대해 절대적으로 움직이게 된다. 일반적으로 absolute를 쓸 경우, 절대적으로 움직이고 싶은 부모에게 position: relative; 를 부여하면 된다.

3)fiexd

: fixed는 말그대로 고정됐다는 뜻이다. absolute는 relative를 가진 부모가 필요했는데, fixed는 필요없다.

.coupon {

position: fixed;

right: 0;

bottom: 0;

background-color: red;

color: white;

font-size: 20px;

}

fiexd 속성값으로 웹브라우저 하단에 고정됨

2.iline, inline-block,block에 대해서

1)block

: block 요소의 의미는, 이 요소 바로 옆(좌우측)에 다른 요소를 붙여넣을 수 없다는 뜻입니다. <div>, <p>

​

2)inline

: inline이라는 말 그대로 inline 요소는 요소끼리 서로 한 줄에, 바로 옆에 위치할 수 있다는 뜻입니다. block 요소와 성질이 반대인

<span>, <a>, <img> 태그 등이 inline 요소입니다.

​

3)inline-block

block 요소의 성질을 가진 element를 css를 사용하여 inline 성질을 갖도록 하는 css property이다.

(예) <p> : block element

<style>

p{

display : inline-block;

}

</style>

​

3.float에 대해서

1)float을 사용하는 이유

: 페이지 전체의 레이아웃을 잡을때 중요한 도구로 사용된다.

​

2)float 속성 사용법- float속성의 속성값(left,right,none)

: element의 float 속성값이 left이면 그 요소는 부모 요소의 가장 좌측에 배치된다.

float속성 적용 전

float : left 적용 후

두번째 사진은 float된 element의 부모 element인 .grid의 높이가 0이 되어서 background-color가 깔릴 공간이 사라져버린 것이다.

​

3)float 속성 문제점 - 왜 float속성의 자녀의 부모 element의 높이가 0가 되는가?

​

: float 속성이 왜 탄생했는지 알아보면 답이 있을 것 같다.

float는 본질적으로 가로배치 전용 속성이 아니다. 원래는 아래 사진과 같이 이미지와 내용이 어우러지게 하기 위해서 만들어진 속성이다. .grid-item의 시작점과 contents의 시작점이 같은 것을 알 수 있다. 이것을 보면 알 수 있듯이 float속성이 부여되면 위로 뜨게 되는 것이다. 결론은 부모인 grid 입장에서는 float된 자식 요소는 높이를 반영시키지 못한다는 얘기다. text가 없으면 높이가 없어진다. 하지만 float된 요소는 그림자처럼 부모 element에 영역을 차지하는 것 처럼 보인다. 이 그림자 영역에 대해서 생각해보면 원래 div element는 block element지만 float되면서 마치 inline element가 되는 것 처럼 보인다. 그결과 c 박스가 전체를 차지하던 block element에서 자기 자신의 영역만 차지하는 inline 요소처럼 바뀌어서 옆에 빈공간이 남게되고.. 그 자리를 문자가 채운 것 처럼 보인다.

​

4)해결 방법 - overflow, clear

:

해결방법1. clear

#yellow에 clear 속성을 적용합니다.

clear:left의 속성을 가진 테그는 float:left의 속성의 영향을 받지 않는다.(

​

(1)float속성 적용

float속성을 적용하면 발생되는 일

(2)#yellow에 clear 속성 부여한 후

clear: left를 해주면 왼쪽에 떠있는 float의 영향을 받는 요소(#red, green, blue)를 float 해제시킨다.

그러면 #yellow는 float의 영향을 받지 않으므로 다시 좌, 우측에 다른 요소를 붙일 수 있는 block element의 효과를 갖게 되고 그렇기 때문에 밑으로 내려가게 된다.

해결방법2.

주로 많이 사용하는 방법인데 바깥 div(.wecode-box)에 overflow: hidden; 을 주는 것입니다.

float속성을 가진 element의 부모 element에 overflow 속성을 주는 것이다.

​

해결방법3.

바깥 div(.wecode-box)를 float시킵니다.

그러면 자식의 float 높이를 인지하여 그만큼 높이를 차지하게 됩니다.

하지만 이것도 문제가 있죠, float이 되어버려 block 요소의 성질이 없어지게 됩니다. 이러면 width를 100% 추가하거나 해야합니다.
